About this tool & Using color in scientific plots

Choosing color schemes

When choosing how to present data in a scientific plot, color is essential to communicating ideas effectively. This can include ensuring that the same colors are used to signify the same concepts across all figures in your paper, ensuring that it is easy to tell the difference between different colors in each individual figure, and using colors that people will easily be able to tell apart. It is also useful to use color to allow people to visually group together similar data - for example, all of the data related to branches of one nerve might be in cool colors, while all of the data from branches of another nerve might be in warm colors.

Categorical plots in this tool work by finding a set of random colors and choosing from within that group which ones are most distinguishable in full color vision and for the most common variants of colorblindness. If you generate a new set of colors after locking some subset of colors, the tool will attempt to find the colors that are maximally distinguishable from your locked colors and from each other.

Additionally, it is common for gradient plots to produce visual distortion - so using the wrong gradient plot can introduce artifacts into the display of your data, as shown here. Gradient plots in this tool are generated by finding two random colors and calculating the gradient between them. If you add set points (intermediate locked values) to the spectrum, you can then generate a gradient that passes through those values - if you increase the smoothing, the tool will attempt to pass near those values while also maintaining a smooth curve across the full space.

Types of colorblindness

Globally, about 8% of men and 0.5% of women have some form of colorblindness. There are three types of colorblindness, one for each type of cones in the retina. Protanopia, which is red-blindness, affects about 2% of people. Deuteranopia, which is green-blindness, is the most common and affects about 6% of people. Tritanopia, which is blue-blindness, is very rare.

Of these, people may be completely lacking a cone of a specific color type, meaning it is not possible to perceive that color at all, or they may simply have a low number of cones of a particular type, resulting in less severe colorblindness. It is more common to have partial colorblindness than to be fully unable to perceive a particular color of light. In this tool, I used the simulation matrices generated by Machado et al (linked below) to simulate a total absence of each cone type.

Perceptually uniform colormaps

Generating a gradient does not necessarily mean you have a smooth gradient. The lightness difference between each step of the gradient may not be evenly distributed, for example in a gradient that passes through blue-green-yellow-orange. Additionally, human vision is non-linear; meaning that an equal change in the wavelength of light does not mean a human eye will experience it as equally different color (read more here). Taking these differences into account is the idea behind making perceptually uniform colormaps.

The difference between two different colors can be calculated as a Euclidean distance in a color space, but RGB and HSL are both non-perceptually-uniform color spaces, leading to inconsistently spaced colors. One relatively perceptually-uniform color space is CIELAB (although there have been small non-uniformities discovered). In the CIELAB space, there is a lightness value (L*), a red-green axis (b), and a blue-yellow axis (a). This tool calculates both the distance between color lightnesses (L*) and the Euclidean distance (deltaE) within the CIELAB space as metrics of perceptual uniformity.
